## Package Contract Status

- New service scaffolds now expose local operation metadata that a later
  package-contract sync/generator can lift into `src/attune.package.ts`.
- The scaffold intentionally does not import package-contract builders yet
  because the full package-contract generator is a separate Phase 2 slice.
- `PackageLayer` and `PackageTestLayer` are exported from the service module so
  early generated services have a deterministic layer surface before full
  package-level composition lands.

## Residual Debt

- Existing hand-authored services across the repo are not migrated by this
  slice.
- Existing service files previously generated by `@attune/nx:effect-service`
  are not rewritten by this slice.
- The generated operation schemas are safe placeholders (`Schema.Void`); agents
  adding real public operations must replace them with meaningful input/output
  schemas or let the future package-contract generator/sync pass fill them.
- The generated registration is visible metadata, not the final
  `definePackageContract` entry. The package-contract generator still needs to
  consume or supersede this slot.
- The scaffold does not yet emit typed error schemas, atom/Reactivity views,
  Effect RPC harnesses, property harnesses, or compile-only assertion modules.
- The generated service module exports `PackageLayer` and `PackageTestLayer`
  locally as requested. The current barrel helper still writes `export *`, so a
  package with multiple generated service modules may need the package-contract
  generator or a future barrel policy to avoid re-exporting duplicate
  package-level aliases from `src/effect/services/index.ts`.

## Blockers

- None for this slice.

## Next-Agent Recommendations

- The package-contract generator should lift `<Service>RunOperation` into
  `src/attune.package.ts` or generate a stronger replacement that uses the
  real typed operation builders.
- The package-contract generator should decide whether service-local operation
  metadata remains as the authoring placeholder or becomes a generated
  implementation detail after `src/attune.package.ts` exists.
- The property/RPC harness agents should target `PackageTestLayer` and public
  `Effect.Service` accessors from generated services instead of importing
  private `make<Service>` helpers.
- A later migration agent should scan current `Context.Tag` services and either
  migrate them to this shape or add package-contract waivers where the lower
  level boundary is intentional.
